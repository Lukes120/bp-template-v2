## Cosa cambia

### 1. Riga TOTALI in lista offerte

Vista interessata: dashboard offerte (`screen === "list"`).

- **Sfondo** della riga TOTALI: `slate-600 #475569` (era chiaro, illeggibile)
- **Costi** in rosso chiaro `#fca5a5` (prima `#64748b` grigio sbiadito)
- **Ricavi** in bianco grassetto, **Margine €** in verde chiaro `#86efac`
- **Font**: `.78rem → .95rem`, peso `500 → 700`, padding `9 → 14px`
- **Badge `%`** ingrandito (`.88rem`, padding più generoso)
- Refactor: gli `style=""` inline sostituiti da classi CSS (`.tot-cell-label/-ricavi/-costi/-margine/-mp`)

### 2. Form offerta a tutta larghezza

- `.o-form-view max-width: 1100px → 1600px`
- `padding: 16px → 24px`
- Su Full HD (1920px) i margini laterali passano da ~410px a ~160px per lato

### 3. Tipografia colonna Prezzo Vendita

In tutte le 5 sezioni del form (Manodopera, Materiali, Servizi, Manutenzione, Trasferte):

- **Righe (`.td-pv`)**: `font-weight: 500 → 700` (grassetto), `font-size: .82rem` (riallineato alla base)
- **Riga totale di sezione (`tfoot td`)**: `font-weight: 500 → 700`, `font-size: .82rem → .92rem` (1 punto più grande delle righe — i totali emergono)

### 4. Refactor PDF singolo offerta (`core/pdf.php`)

Problema risolto: il PDF spezzava su 2 pagine sprecando la seconda.

- `@page { margin: 10mm }` (era ~25mm default Dompdf)
- `page-break-inside: avoid` sul blocco Riepilogo finale
- **Salta sezioni vuote**: se totale costo sezione = 0, intera sezione omessa (es. Manutenzione senza voci)
- **Salta righe a zero** dentro le sezioni
- **Salta riga "Sconto Direzione"** se non c'è sconto (prima compariva sempre con 0,00)
- **Palette slate-600** uniforme (era blu acceso `#1d4ed8` / `#1e3a8a`)
- **Header compatto** (logo 50→38px, font titolo 13→12px)
- **Padding celle**: `2px 5px → 1px 4px`, tabelle `margin-bottom: 6→3px`, h3 `8px 0 3px → 5px 0 2px`
- **Zebra striping** leggera sulle righe
- **Footer centrato** con utente + data + branding

### 5. Refactor Excel singolo offerta (`core/excel.php`)

- **Palette slate-600 / slate-500** (era `#1A3E5C` / `#2E6DA4`)
- **Salta righe a zero** in tutte le sezioni
- **Page setup**: A4 landscape + **fit-to-page-width = 1** → la stampa sta in una pagina larga
- **Conditional formatting** sulla cella margine % finale (verde ≥25%, giallo 15-24%, rosso <15%)
- **Header/Footer di stampa** con titolo + utente + numero pagine
- **Margini** ridotti a 0.4"
- Firma estesa: `bp_xlsx_offerta(array $form, ?string $utente = null)` (backwards-compatible)

### 6. Refactor PDF archivio (`bp_pdf_archivio`)

- **KPI bar** in cima al PDF (5 box: Offerte, Ricavi, Costi, Margine €, Margine % con semaforo)
- **Palette slate-600** uniforme
- **Costi** delle righe in rosso `#b91c1c` (coerente con riga totali dashboard)
- `@page { margin: 12mm }`

### 7. Refactor Excel archivio (`bp_xlsx_archivio`)

- **Freeze pane** alla riga 4 (intestazioni fisse durante scroll)
- **Auto-filter** su tutte le colonne dati (frecce filtro Excel native)
- **Conditional formatting** colonna margine % (semaforo)
- **Gridlines off**
- **Page setup** + rows-to-repeat-at-top (intestazione ripetuta per pagina di stampa)
- **Header/Footer** stampa con N. offerte e numerazione pagine

### 8. Allega a Odoo

Nessuna modifica diretta — `api/allega_odoo.php` chiama le stesse funzioni `bp_pdf_offerta()` e `bp_xlsx_offerta()` aggiornate. I file caricati come `ir.attachment` su Odoo sono **bit-per-bit identici** ai download dalla dashboard.

## Deploy sulla VM

### Pacchetto pronto

`C:\Users\lranalletta\Documents\bp-template-v2-update.zip` (~112 KB)

Esclusi: `data/`, `vendor/`, `credentials.env`, `*.bak`, `*.JPG` di esempio, `pdf.pdf`, `excel.xlsx`.

### Step

#### 1) Backup pre-deploy sulla VM

In RDP su `UTWGENWEB02` (10.1.2.122), PowerShell come admin:

```powershell
$src='C:\laragon\www\bp-template-v2'
$bak="C:\Users\Administrator\Documents\bp-template-v2-backup_$(Get-Date -Format 'yyyyMMdd_HHmm').zip"
Compress-Archive -Path "$src\*" -DestinationPath $bak -Force
Get-Item $bak | Select-Object Name, Length, LastWriteTime
```

#### 2) Trasferisci lo zip sulla VM

Drive condiviso RDP: da Esplora risorse della VM, copia da `\\tsclient\C\Users\lranalletta\Documents\bp-template-v2-update.zip` a `C:\Users\Administrator\Documents\` (o dove preferisci).

#### 3) Estrai

```powershell
$zip='C:\Users\Administrator\Documents\bp-template-v2-update.zip'
$dst='C:\laragon\www\bp-template-v2'
Expand-Archive -Path $zip -DestinationPath $dst -Force
```

`-Force` sovrascrive solo i file presenti nello zip. `data/`, `vendor/`, `config/credentials.env` non vengono toccati.

#### 4) Niente da fare lato server

- No `composer install` (composer.json invariato)
- No restart Apache (PHP module, ricarica file ad ogni request)
- No migrazione DB (schema invariato)
